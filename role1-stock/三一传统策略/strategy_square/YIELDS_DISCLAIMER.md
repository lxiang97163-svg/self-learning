## 1. 数据性质

- 所有出现的收益率数字，均为同花顺策略广场页面的**历史回测数据**（backtest），
  由同花顺服务器基于历史行情、按策略问句对应的回测模型计算得出。
- **不是实盘账户的真实交易收益**。
- **不是作者本人或本项目复盘过的收益**。
- 回测模型内部的假设（滑点、手续费、成交规则、停牌处理等）均由同花顺端决定，
  本项目无法完整知晓，也无法为其真实性背书。

## 2. 本项目与这些数字的关系

- 本项目只做"爬取 + 结构化 + 问句筛选"，不对数字做任何二次计算。
- 本项目**不反向推销**任何一条策略。列出某条问句收益率高，不代表推荐使用。
- 本项目的筛选维度是"逻辑是否符合三一/连板/炸板/先人指路等体系"，
  收益率仅作为**辅助参考字段**。

## 3. 历史回测 ≠ 实盘

即使某条策略的回测历史收益率非常高，用户**在实盘使用时依然可能亏损**，原因包括但不限于：

- **未来函数污染**：部分"漂亮"的历史回测问句可能隐式依赖未来信息，
  回测走得好，实盘根本无法复现。本项目 Task C 已剔除了 22 条显性未来函数问句，
  但**不能保证覆盖所有隐性未来函数**。
- **冲击成本**：回测默认按收盘价或开盘价成交，实盘存在滑点、流动性冲击。
- **市场环境差异**：某一时期的回测表现依赖当时的市场风格、情绪、政策环境，
  风格切换后策略可能彻底失效。
- **幸存者偏差**：策略广场上排序靠前的策略，已经是"被选出来的好看的"，
  其整体分布远好于全量策略。
- **参数过拟合**：问句里隐含的阈值（涨幅 3%、竞价成交额 3000 万等）可能是
  对历史数据过拟合的结果。

## 4. 用户采用这些问句前的强制流程

1. **自行复测**：用自己的回测框架（或同花顺其他入口）在**不同时间窗口**重跑一次，
   观察收益分布的稳定性，而不只看一个总数。
2. **剔除未来函数**：人工再过一遍问句文字，确认没有"次日 / 未来 / 下周"之类的
   词语混入（Task C 已筛一遍，但请**再人工 review 一遍**）。
3. **小仓位实盘验证**：任何策略在重仓使用前，必须以 **≤1% 总仓位**跑至少 1 个月
   实盘，观察是否与回测一致。
4. **止损纪律**：永远先想"这策略错的时候我最多亏多少"，再想"对的时候能赚多少"。
