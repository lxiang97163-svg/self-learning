## 2. 方法论

### 2.1 采集（Playwright 备用脚本）

- 目标页：`https://backtest.10jqka.com.cn/backtest/app.html#/strategysquare`
- 使用 `playwright-python` 无头 Chromium，模拟真实浏览器渲染 SPA。
- 抽取逻辑：用启发式选择器 `[class*="strategy"], [class*="card"], [class*="item"], [class*="question"], [class*="strat"]` 捞所有卡片 DOM；再用 JS 对每个卡片做「首行 = title、最长行 = question、class 含 tag/label/badge 的子元素 = tags」的拆分。
- **新增 · 收益率字段抽取**（方案 3 的 v2 升级）：
  - Python 侧对每张卡片的 `fullText` 用正则+多组标签启发式匹配：
    - `yield_30d`：近30日收益率 / 近30天收益率 / 近30日 / 30日收益 / 30天收益率 / 月收益率
    - `yield_180d`：近180日收益率 / 近半年收益率 / 近180天收益率 / 180日收益 / 半年收益率
    - `yield_1y`：近1年收益率 / 近一年收益率 / 近12个月收益率 / 年化收益率 / 1年收益
    - `win_rate`：胜率 / 策略胜率 / 成功率
    - `max_drawdown`：最大回撤 / 回撤 / 最大回撤率
  - 任何字段失败返回 `None`，**不编造数字**。
- 翻页：尝试 `button:has-text("下一页")` 等多组选择器，默认最多翻 5 页，硬上限 10 页。
- 反爬礼貌：请求间隔 ≥2.0s + 0~0.6s 随机抖动（比 v1 的 1.2s 更保守）；UA 伪装为普通 Chrome；未并发。

### 2.2 筛选（自动剔除 + 加分保留）

**自动剔除关键字**（出现即剔除）：
- 未来函数：`次日`、`未来`、`下一交易日的收盘价/最高价/最低价`、`本周后`、`T+N 日收盘`
- 末态（盘后才知，竞价时无法使用）：`当日收盘涨停`（但保留「昨日涨停」「昨日触板」「竞价涨停」）
- 长线基本面：`PE`、`市盈率`、`股息率`、`ROE`、`净资产收益率`、`业绩增长率`、`PB`、`市净率`
- 无逻辑筛股：`随机选取`、`字母顺序`、`股票代码以X开头`
- 用户明确过滤项：`ST`、`北交所`、`上市天数<30`（保留则剔除）

**加分保留关键字**（命中多个优先保留）：
- 竞价类：`竞价`、`集合竞价`、`开盘成交额`、`竞价换手率`、`竞价涨幅`
- 连板类：`连板`、`连续涨停`、`封单`、`封板时间`、`炸板`、`开板次数`
- 题材/排序：`概念板块内排名`、`题材龙头`、`所属板块涨幅`
- 资金面：`主力资金`、`龙虎榜`、`净买入`

- 风控：`非ST`、`非北交所`、`上市天数`、`流通市值`、`未停牌`

## 3. 关于策略收益率（重要）

### 3.1 数据来源

策略广场每条策略在同花顺官方页面上**通常**会展示以下回测字段：

| 字段 | 含义 | 采集脚本对应字段 |
|------|------|-------------------|
| 近30日收益率 | 过去 30 个自然日的策略组合回测收益 | `yield_30d` |
| 近180日 / 半年收益率 | 过去 180 天的策略组合回测收益 | `yield_180d` |
| 近1年 / 年化收益率 | 过去 1 年的策略组合回测收益 | `yield_1y` |
| 胜率 | 回测区间内盈利次数占比 | `win_rate` |
| 最大回撤 | 回测区间内最大的阶段性亏损 | `max_drawdown` |

这些都是**同花顺服务器基于历史行情计算的回测数据**，**不是实盘**。

### 3.2 三种数据来源状态（本项目 source 字段口径）

`strategies_raw.json` 与 `yields.json` 里每条记录都会带 `source` 字段，共三种：

| source 值 | 含义 | 本次交付中的出现情况 |
|-----------|------|----------------------|
| `页面抓取` | Playwright 脚本从策略广场真实渲染页面抽到的数字 | **未出现**（WebFetch 被拒，脚本需用户本地跑） |
| `经验估算` | 按问句类别给出的经验范围（带 `[估算]` 标注） | **未出现**（为遵守"严禁编造数字"约束，本次放弃了方案 4） |
| *留空 + note* | 未能抓取，需用户手动跑脚本 | **当前所有 48 条的状态** |

### 3.3 估算 vs 实测的区别

- **实测（页面抓取）**：数字为同花顺回测引擎输出，精确到小数点后 2 位，但**仍然不是实盘**。
- **经验估算**：本项目**本次未使用**。如果未来启用，将只给"类别范围"（例如"首板三一类近 30 日 0~15%"），**不给具体数字**，且必须带 `[估算]` 前缀。

### 3.4 注意事项（强制阅读）

1. **回测 ≠ 实盘**。滑点、冲击成本、流动性、停牌处理、分红除权、手续费，在同花顺服务器回测里的处理方式可能与真实成交完全不同。
2. **幸存者偏差**。策略广场排序靠前的策略已经是"被人为或算法选出来的好看的"。
3. **参数过拟合**。问句中隐含的阈值（涨幅 3%、成交额 5000 万等）本身可能就是回测拟合结果。
4. **隐性未来函数**。Task C 已剔除 22 条显性未来函数，但无法保证剩余 48 条不含隐性未来函数，**用户必须再人工 review 一遍**。
5. **市场环境切换**。某一时期的回测表现依赖当时的市场风格、情绪、政策。风格一换可能彻底失效。

## 5. 用户下一步操作

### 5.1 立即可用（不需要采集）

直接打开 `strategies_filtered.md` 使用 48 条保留问句。粘贴到 <https://www.iwencai.com> 搜索框即可验证。收益率列此时为空。

### 5.2 真实采集策略广场收益率数据

```powershell
# 在项目任意 Python 虚拟环境中
pip install playwright==1.47.0
playwright install chromium

cd "E:/李响git工作空间/self-learning/role1-stock/三一传统策略/strategy_square"
python collect_strategies.py --pages 5
```

运行成功后，以下两个文件会被脚本同时覆写：

- `strategies_raw.json`：结构为 `{collected_at, source, disclaimer, items: [{title, question, tags, author, popularity, yield_30d, yield_180d, yield_1y, win_rate, max_drawdown, source}, ...]}`
- `yields.json`：结构为 `{collected_at, source, disclaimer, items: [{title, question, yield_30d, yield_180d, yield_1y, win_rate, max_drawdown, source, disclaimer}, ...]}`（仅收益率相关字段，便于与 `strategies_filtered.md` 做 join）

然后可将 raw 结果再次按 `strategies_filtered.md` 中「2.2 筛选」的关键字规则人工/脚本二次过滤，并手工把对应问句的收益率填回 `strategies_filtered.md` 的"收益率(近30日/180日/1年)"列。

### 5.3 与现有项目联动

可将 `strategies_filtered.md` A 区的组合推荐作为 `三一传统策略/竞价三一_断板弱转强.py` 或 `一红+爆量.py` 的选股预筛入口，例如用 "A1#1 + A7#44" 组合在盘前 9:20-9:25 生成候选池。
