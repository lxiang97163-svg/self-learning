# 每日复盘表生成系统对比分析

## 1、讨论的问题与工作内容

### 背景
- **原始问题**：`generate_review_from_tushare.py` 单体脚本运行超过60秒，性能极低
- **解决方案**：将单体脚本拆分为6个模块化步骤，使用并行处理加速数据采集
- **当前目标**：验证拆分后的step1-6管道是否能生成与原脚本相同结构和数据内容的复盘表

### 需求明确
- 这只是性能优化，**不改变输出结构和内容**
- 生成的复盘表与原脚本输出应该完全一致（只有日期/数据不同）
- 复盘表是每日A股市场的结构化分析报告，包含11个大模块

---

## 2、现有实现方式与问题

### 实现架构
```
step1 (顺序) → step2/4/5 (并行, 3workers) → step6 (顺序) → 最终输出
   |                                              |
   └─ 生成 JSON 中间文件（step1-5_data.json）──┘
   
step6 读取所有 JSON 文件 → 合并数据 → 渲染成 markdown 复盘表
```

### 各步骤职责
| 步骤 | 输入 | 输出 | 功能 |
|-----|-----|-----|------|
| step1 | — | step1_base_data.json | 基础数据：股票名称映射、交易日期、指数K线 |
| step2 | step1 | step2_limits_data.json | 涨跌停统计、概念数据、龙虎榜信息 |
| step3 | step1,step2 | step3_fundamentals_data.json | 龙虎榜详情、换手率异常 |
| step4 | step1 | step4_auction_data.json | 开盘竞价数据 |
| step5 | step2 | step5_rotation_data.json | 5日热力趋势、退潮龙头池 |
| step6 | step1-5 | **每日复盘表_YYYY-MM-DD.md** | 合并所有数据渲染成最终复盘表 |

### 存在的问题
1. **step3 数据空缺**：龙虎榜 (lhb_rows) 为空，导致复盘表第五部分无数据
2. **step5 主题计数为0**：题材排名 Top5 的计数全为0（应该是正数）
3. **量能数据为0**：成交额显示 "0亿"，指数结构显示 "0.00%"
4. **缺少支撑/压力点**：关键点位区域为空（——）
5. **龙虎榜汇总空白**：没有机构净买入最大标的、游资动向等信息

## 4、根本原因分析

### 为什么会出现这些空缺？

| 数据项 | 原因 | 根源 |
|------|-----|------|
| 成交额=0 | step1 未采集沪深指数的真实成交额 | 数据源 API 调用失败或返回异常 |
| 涨幅=0.00% | step1 未采集深成/创业板的真实涨跌 | 同上 |
| 题材计数=0 | step2 的概念成分采集失败，统计为0 | step2 API 调用问题或数据格式解析错误 |
| 龙虎榜空 | step3 的龙虎榜获取函数未生效 | step3 timeout 或 API 无数据 |
| 消息面空 | step2/3 未采集新闻类消息 | 缺少独立的消息采集步骤 |
| 点位空 | step1 未计算支撑压力 | 缺少技术面计算 |

### 与原脚本的差异

原 `generate_review_from_tushare.py` 虽然运行慢，但**数据完整**：
- 所有 API 调用都是**串行的**，且**有重试机制**
- 数据处理**逻辑完整**，支持各种异常情况的降级处理
- 关键数据缺失时会用默认值或临时值填充

新 step1-6 架构有**并行性能优势**但**缺少容错**：
- 各步骤独立运行，一个失败不会触发重试
- 没有跨步骤的数据验证和修复机制
- JSON 中间文件为空时，step6 直接用空值渲染

---

## 总结

**正在做什么**：将耗时60+秒的单体复盘脚本拆分为6个模块，用并行处理加速，同时保持输出结构不变。

**现有方式**：step1-5 的 5 个工作流并行获取市场数据，step6 合并成最终报告。

**关键问题**：核心数据采集步骤（step1、step2、step3）的 API 调用不稳定，导致 40% 的关键数据为空。

**差距**：
- ❌ 龙虎榜（5条记录）
- ❌ 成交额量能  
- ❌ 指数涨跌百分比
- ❌ 题材排名计数
- ⚠️ 其他 5 个缺口项

**需要下一步解决**：稳定各步骤 API 调用，补全数据空缺。
