# 一进二数据导出 — 项目概况（`init.md`）

> 本文面向 **人类接手者** 与 **AI 助手**：快速理解本文件夹在做什么、数据从哪来、规则怎么变、改代码时要注意什么。  
> 与业务定稿一致的书面规则见同目录 **`一进二选股规则.md`**（由脚本生成，勿手改核心数字除非有意重跑）。

---

## 1. 背景是什么

- **策略主题**：A 股「首板/二板/三板 → 次日竞价」的一进二风格筛选。候选池为 **T-1 日「严格首板 ∪ 严格二板 ∪ 三板」**（主板、非 ST），特征以 **T 日 9:25 竞价** 为主，并叠加 **T-1 日开板次数**（`limit_list_d` 可复核），用于宽松包络硬规则、**反例收紧**与逆向分析。
- **为何单独成文件夹**：本目录既存放 **可执行脚本**（拉数、选股、逆向），也存放 **导出的原始与分析 CSV**，以及 **书面定稿规则**，属于同一套逻辑的「项目包」。
- **接口依赖**：`chinadata.ca_data` / `chinamindata.min`（类 Tushare 用法），需有效 token；具体密钥在脚本内配置，**勿提交到公开仓库**。
- **当前代码状态**：`构建一进二选股规则.py`、`928_prepare.py`、`928_run.py`、`928选股_cursor.py` 已按 **首板∪二板∪三板联合池 + 6 条锚定样本（含合力泰）+ 宽松包络（±30%/50%）+ 反例验证日收紧 + auction_price_ratio ≥ 1.0 + 原型排序** 对齐，需重跑 `构建一进二选股规则.py` 生成新版 `rule_params.json`。`928选股_cc.py` 仍保留旧版固定参数逻辑作为备用。`backtest_cursor_1month.py` 默认区间改为 04-01~04-20。

## 3. 思路是什么

1. **池子先定死**：在 **T-1** 涨停列表中筛 **严格首板**（`limit_times=1`、T-2 未涨停）、**严格二板**（`limit_times=2`）与 **三板**（`limit_times=3`），三路 **并集**，限制 **主板、非 ST**。竞价与池内名次均在 **该联合池** 内计算。
2. **竞价**：仍用接口可复核字段（涨幅、成交额、换手率等）；排名均在 **当日联合池内**。
3. **T-1 质量/盘口**：从 `limit_list_d`（及后续若接入的明细接口）抽取对逆向有帮助的列，例如 **首次封板时间、开板次数、封单金额/强度** 等，进入宽表；是否进入 **硬规则包络** 由 `derive_params` 扩展列清单决定（可与竞价阈值分层：先竞价硬过滤，再质量分排序）。
4. **跨日排名用占比**：用「池内名次 / 池内有效样本数」而非绝对名次，减少池子大小变化带来的失真。
5. **参数来自宽松包络 + 反例收紧**：不再使用「紧包络」（样本 min/max 直接取边界），而是在样本 min/max 基础上分别放宽 30%（下界）和 50%（上界），避免对少数样本过拟合。`fd_amount_min` / `first_time_max` / `turn_max` 已删除（过拟合、不通用）；`auction_price_ratio_min` 固定为逻辑门槛 **1.0**（今日竞价价须高于昨日）。推导后对**反例验证日**（`NEGATIVE_DATES`）进行贪心收紧。最终参数导出为 `rule_params.json`。
6. **新高约束**：锚定样本均满足竞价创新高时，规则层开启 `need_new_high`（见定稿 md）。
7. **多票时排序**：先硬过滤，再按与参考样本 **原型（中位数中心）** 的距离排序；实盘另有 `928选股_cursor.py` 可单独跑当日。

## 5. 分析过程是什么

1. **取数**：按 `FETCH_DATES`（需覆盖所有锚定日与反例日，例如含 `20260410`、`20260413`、`20260414`、`20260415`、`20260416`、`20260417`）逐日 `collect_day` → `build_df`（合并 pool、竞价、市值、历史特征、新高标记、**T-1 涨停质量字段**等）。
2. **参考行抽取**：`REFERENCE_CASES = ANCHORS`；每条都必须在对应日宽表里存在且关键字段非空。
3. **derive_params**：对参考样本做 min/max + round，得到竞价侧 `pct_min/max`、`amount_min`、`turn_min`、三名次占比上限、`prev5/10` 上限、`need_new_high`；质量扩展项仅在 **全锚定样本该字段均非空** 时才纳入硬规则。
4. **反例收紧（`tighten_for_negatives`）**：对 `NEGATIVE_DATES`（04-14、04-16）回放，若仍有命中则贪心逐步收紧参数，每轮选减少最多反例且不丢锚定的维度。
5. **回放与统计**：对锚定日与反例日计算命中列表、收盘相对竞价等（用于文档说明，非网格搜索）。
5. **定稿落盘**：`一进二选股规则.md`；CSV 与脚本 **同目录**（`OUT_CSV_DIR = DIR`）。

若 **新增锚点或改日期**：需改 `构建一进二选股规则.py` 中的 `ANCHORS` / `FETCH_DATES` / `REFERENCE_CASES` / `NEGATIVE_DATES`，然后重跑生成 md、`rule_params.json` 与缓存。`928_prepare.py` 和 `928选股_cursor.py` 中的 `ANCHORS` / `REFERENCE_CASES` 也需同步（降级用）。

## 6. 包含哪些数据、获取方法是什么

### 6.1 目录内文件命名（CSV）

含义见 `README_字段说明.csv`。当前重跑后的主文件前缀已使用能体现 **首板∪二板联合池** 的命名，例如 `YYYYMMDD_01_首板二板联合池_limit_list.csv`。

| 前缀（规划） | 含义 |
|--------------|------|
| `YYYYMMDD_01_…联合池` | T-1 `limit_list_d` 筛出的 strict 首板 ∪ strict 二板 + 主板 + 非 ST + 质量字段 |
| `YYYYMMDD_02_竞价原始` | T 日 `stk_auction` 池内标的 |
| `YYYYMMDD_03_前日市值` | T-1 `daily_basic`（流通市值等） |
| `YYYYMMDD_04_历史特征` | 并行 `daily` 汇总的 prev_5d/prev_10d/high20/high60 |
| `YYYYMMDD_05_当日收盘` | T 日 `daily` 池内（用于相对竞价收益） |
| `YYYYMMDD_06_分析宽表` | 合并 + 名次与名次占比 + 收盘与相对竞价 + **T-1 涨停质量** |
| `00_分析宽表_全部日期合并` | 各日 06 纵向合并 |
| `一进二选股规则.md` | 书面定稿（生成物） |
| `rule_params.json` | 收紧后的参数与原型（供 928_prepare / 928选股_cursor 直接读取） |
| `init.md` | 本说明（人工维护 + 可随项目迭代更新） |

### 6.2 获取方式（技术要点）

- **联合池**：`limit_list_d`（涨停列表）按 **首板** 与 **二板** 规则分别筛选后 **去重并集**；二板判定需与 `limit_times`、前序交易日涨停状态一致（实现时写在 `collect_day` 一处）。
- **竞价**：`stk_auction`（分钟/竞价接口封装在 `pro_min`）。
- **T-1 涨停质量**：`limit_list_d` 中已有字段优先（如首次封板时间、开板次数等）；若需封单额等，再评估是否增加接口或字段映射。
- **市值**：`daily_basic` 按代码并行。
- **历史**：`daily` 按代码并行，算 5/10 日涨幅与 20/60 日最高价。
- **收盘**：`fetch_close_pool_parallel` 仅池内按代码请求，避免全市场 `daily(trade_date)`。

## 7. 结果是什么（与 `一进二选股规则.md` 对齐）

当前口径已不是旧版「仅首板池」，而是 **首板∪二板联合池 + 6 条锚定样本 + 反例收紧 + 质量字段硬规则 + 原型排序**。以最近一次重跑生成的 `一进二选股规则.md` 为准。

### 7.1 最新量化标准

适用范围：T 日 9:25，对 **T-1 严格首板 ∪ 严格二板联合池** 进行过滤与排序。

| 参数 | 说明 | 推导方式 |
|------|------|---------|
| `pct_min` | 竞价涨幅下限（%） | 样本最小值 × 0.70，按 0.1 取整 |
| `pct_max` | 竞价涨幅上限（%） | 样本最大值 × 1.50，按 0.1 取整 |
| `amount_min` | 竞价成交额下限（亿元） | 样本最小值 × 0.70，按 0.01 取整 |
| `turn_min` | 竞价换手率下限（%） | 样本最小值 × 0.70，按 0.01 取整 |
| `pct_rank_ratio_max` | 竞价涨幅名次占比上限 | 样本最大值 × 1.50，按 0.01 取整 |
| `turn_rank_ratio_max` | 竞价换手名次占比上限 | 样本最大值 × 1.50，按 0.01 取整 |
| `amt_rank_ratio_max` | 竞价成交额名次占比上限 | 样本最大值 × 1.50，按 0.01 取整 |
| `prev5_max` | 近 5 日涨幅上限（%） | 样本最大值 × 1.50，取整 |
| `prev10_max` | 近 10 日涨幅上限（%） | 样本最大值 × 1.50，取整 |
| `need_new_high` | 竞价价须不低于近 20 或近 60 日最高价 | 当全部锚定样本均满足时生效 |
| `open_times_max` | T-1 开板次数上限 | 样本最大值 + 2（缓冲） |
| `auction_price_ratio_min` | 今日竞价价 / 昨日竞价价下限 | **固定 1.0**（逻辑门槛，不从样本推导） |

已删除：`turn_max`（过拟合）、`fd_amount_min`（过拟合）、`first_time_max`（过拟合）。

具体数值以最新一次重跑 `构建一进二选股规则.py` 的输出（`rule_params.json`）为准。

### 7.2 当前样本内覆盖结果

- 参考样本覆盖：**6 / 6 命中**（含合力泰 04-20，收紧后仍全部保留）
- 反例验证日命中：04-14 = 0 只 ✓，04-16 = 0 只 ✓
- 扩展验证：`backtest_cursor_1month.py` 默认回测 04-01~04-20 全部交易日
- 具体命中列表以最新一次重跑 `构建一进二选股规则.py` 生成的 `一进二选股规则.md` 为准

## 8. 结论是什么

- **高置信部分**：联合池定义、竞价字段、池内名次/占比、以及当前纳入的 T-1 质量字段，均已由脚本与导出宽表复核。反例验证日（04-14、04-16）确认命中 0 只。
- **中低置信部分**：阈值集合来自 **当前锚定包络 + 反例贪心收紧**，市场结构变化后可能出现空池或多票，需 **增删锚点/反例日并重跑包络**，或调整 `ROUND_STEP` 与特征集。
- **与旧版 928**：`928选股_cc.py` 为另一套固定参数与旧口径备用脚本，与 Cursor 规则版 **不等价**；当前以 `一进二选股规则.md`、`928_prepare.py`、`928_run.py`、`928选股_cursor.py` 作为同一套 Cursor 口径。

## 9. 接手后如何改（Checklist）

1. **改锚点/日期/反例**：优先同步 `构建一进二选股规则.py` 中的 `ANCHORS` / `FETCH_DATES` / `REFERENCE_CASES` / `NEGATIVE_DATES`，再重跑 `构建一进二选股规则.py`（会自动收紧并导出 `rule_params.json`）。`928_prepare.py`、`928选股_cursor.py` 中的 `ANCHORS` / `REFERENCE_CASES` 也需同步（降级用）。
2. **改数据口径**：优先改 `first_board_pool_data.py` 与宽表构建函数，避免在多个脚本重复复制竞价/池子/质量字段逻辑。
3. **只更新说明**：改 **`init.md`**（背景/流程）；**数字定稿**仍以重跑生成的 **`一进二选股规则.md`** 为准。
4. **Token 与安全**：勿将 token 推送到公开 git；建议后续改为环境变量。
