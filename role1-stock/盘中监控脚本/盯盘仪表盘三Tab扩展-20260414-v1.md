# 盯盘仪表盘三Tab扩展

> 日期：2026-04-14
> 版本：v1

---

## 本次在做什么

在已有的「竞价实时监控」仪表盘基础上，扩展为一个完整的「复盘→预案→实盘」三Tab网站，将每日复盘表、速查预案和早盘竞价监控串联在同一个Web界面内，支持历史日期回看。

起点：`jumpingnow_all/盘中监控脚本/` 下已有可运行的竞价监控系统（Python + Node.js + HTML）。

---

## 当前结论

- **Tab日期逻辑（已确认）**：选择日期 D 时
  - 每日复盘 → `每日复盘表_{D-1}.md`（前一交易日收盘后写的复盘）
  - 明日预案 → `速查_D.md`（D 日当天的早盘预案，由 D-1 复盘后生成）
  - 竞价监控 → D 日的实时流（今日）或历史快照（历史日）
- **日期选择器范围** = 所有存在 `速查_D.md` 的日期
- **历史竞价快照** = Python 写出 `dashboard_YYYYMMDD.json` 供历史回看

## 未解决事项分析

### 1. 三Tab + 日期选择器（index.html）

- **现状**：当前 index.html 只有竞价监控一个页面
- **思路**：顶部加 Tab 导航栏，右上角加日期下拉（只列有 `速查_D.md` 的日期），切换 Tab/日期时请求对应 API 并重新渲染
- **方案**：纯前端状态机（`currentTab` + `currentDate`），每次切换调 fetch，无需引入前端框架
- **建议**：单文件 HTML + 内联 JS，与现有代码风格一致，改动可控

### 2. 每日复盘解析（server.js + index.html）

- **现状**：复盘表 Markdown 约 760 行，结构化程度高
- **思路**：服务端用正则提取关键段落（8节内容），返回结构化 JSON；前端按卡片布局渲染
- **关键段落**：
  - 指数复盘：开/高/低/收、支撑/压力、量能、预期
  - 情绪复盘（3.2）：风标1/2、情绪节点、涨停/跌停数
  - 情绪日历（3.3）：近5日 date/state/trend
  - 热力追踪（3.5）：前5活跃题材 + 趋势
  - 如果则执行（8.x）：已触发条目
  - 四块打分（10.1）：总分
  - 全文总结：一句话 + 明日关注点 + 操作关键词
- **建议方案**：服务端正则解析 → JSON → 前端渲染卡片。避免引入 `marked` 等依赖，保持零新增依赖

### 3. 明日预案静态展示（server.js + index.html）

- **现状**：速查解析已有 Python 版本，但 JS 端未实现
- **思路**：在 server.js 用正则提取链条/阈值/健康度/B备选/如果则执行，返回 JSON；前端用与竞价监控相同的卡片组件渲染，但不显示实时涨幅
- **方案**：复用前端 `stockRowHtml`、`healthBadgeHtml` 等函数，传入静态数据（pct=null，只显示阈值要求）

### 4. 历史快照（speedcard_monitor.py + server.js）

- **现状**：只保留 `dashboard_latest.json`，历史数据会被覆盖
- **思路**：Python 每次写 latest 时同时写 `dashboard_YYYYMMDD.json`（按日期，只保留最后一次，即「当日最后一分钟快照」）
- **改动**：`_write_dashboard_json` 函数额外写一次带日期的文件（约2行代码）
- **服务端**：新增 `/api/snapshot?date=YYYYMMDD`，优先读日期文件，找不到则返回 latest
