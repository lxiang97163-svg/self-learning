# System Overview — tutor-skills

---

## 这套 skill 解决什么问题

拿到一个新的知识源（文档/代码库）之后，你不知道该怎么系统地学习它，也不知道自己哪里还没掌握。tutor-skills 把这个过程拆成两步：

```
知识源 ──tutor-setup──▶ StudyVault ──tutor──▶ 掌握度追踪 ──▶ 出师
```

## 两个 skill 的职责

```
┌─────────────────────────────────────────────────────┐
│  tutor-setup                                        │
│  职责：把原始材料转成结构化学习材料                     │
│  输入：PDF / 文档 / 代码仓库 / URL                    │
│  输出：StudyVault/ 目录（笔记 + 练习题 + Dashboard）  │
└──────────────────────┬──────────────────────────────┘
                       │ StudyVault/
                       ▼
┌─────────────────────────────────────────────────────┐
│  tutor                                              │
│  职责：通过测验发现盲点，追踪每个概念的掌握程度          │
│  输入：StudyVault/ 目录                              │
│  输出：更新后的 Dashboard + concepts/ 文件            │
└─────────────────────────────────────────────────────┘
```

## StudyVault 内部结构

```
StudyVault/
├── *dashboard*.md          ← 聚合数字：各区域掌握度表 + 统计
├── concepts/
│   ├── {area-name}.md      ← 每个区域的概念追踪（答题记录 + 错误笔记）
│   └── ...
└── {内容笔记目录}/          ← tutor-setup 生成的学习内容
```

## 设计原则

1. **主动回忆优先**：tutor 永远是让你产出（答题），而不是让你读材料
2. **概念级追踪**：追踪的粒度是每个具体概念，而不是笼统的"这章学完了"
3. **CWD 边界**：tutor-setup 只能读写 CWD 内的文件，不能跨目录访问
