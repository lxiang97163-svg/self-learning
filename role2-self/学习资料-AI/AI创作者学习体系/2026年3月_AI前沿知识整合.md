### 1.2 Agent 生态与 MCP

- **MCP（Model Context Protocol）**：Anthropic 发起，现归 Linux Agentic AI Foundation，已被 OpenAI、Google、Microsoft、Amazon 采用，成为「AI 的 USB-C」。
- **Claude Marketplace**（3月7日）：企业采购模式，对创作者零抽成，与 GPT Store 约 $0.03/对话形成对比。
- **Cursor + MCP**：支持 Linear、Postgres、GitHub、Notion 等，可解决「上下文墙」；但连接过多工具有 token 膨胀与成本问题，需要 `.cursorrules` 约束使用。

### 1.3 国内 AI 应用

- **千问**：MAU 2.03 亿，增速 552%，全球增速最高 AI 应用（2026年3月）。
- **豆包与千问**：国内形成双雄格局。

## 二、一人企业 & 创作者经济（商业层）

### 2.1 Dan Koe 核心观点（3月 Newsletter + 播客）

**2026 一人企业本质**：
- 仍要：获客、写内容、做产品、做落地页、写销售文案——但 AI 让你更快、更好、更少试错。
- **门槛更低，但技能天花板更高**：很多人失败是因为不迭代、不学习，指望 AI 全包。
- **Agent 滥用**：大量人烧钱用 OpenClaw、Claude Code 看 Agent 跑，却无实际成果；不是工具问题，是「不理解自己在做什么」。

**$100 万/年 的可行路径**（单人不雇人）：
- 每天卖 18 个 $150 产品（如课程）
- 每天卖 111 个 $25 订阅（如 Substack）
- 每 2 天签 1 个 $5000 客户（自由职业）
- 每 4 天签 1 个 $10000 客户（教练/咨询）
- 或多种组合

**内容生态**：Newsletter → YouTube → 播客 → 帖子 → Reels/Shorts → 轮播图，一源多用。

**你即利基**：不是「写作为利基」，而是你的信念、经历、兴趣交汇处；需要长文载体（Newsletter/视频）才能承载多元兴趣。

**Meaning Economy**：AI 替代更多劳动后，价值交换不会消失；人类仍需要「帮助他人并得到反馈」来获得意义。

### 2.2 Pieter Levels & Marc Lou 模式

- **Pieter**：Photo AI ~$102K/月、Interior AI ~$39K/月、Remote OK ~$36K/月、Nomad List ~$15K/月；「Vibe Coding」用 Cursor 3 小时做出 3D 飞行游戏，17 天内年收入破百万。
- **Marc Lou**：ShipFast（Next.js 模板）>$50K/月；23 个网站实验，2 周一产品，不涨就砍。
- **共性**：无融资、无团队、快速迭代、AI 提效、多产品组合。

### 2.3 AI-Nate 实践

- 17 周 17 个 AI 产品：Momento AI、Builder AI、Creator AI、Research AI、Twain AI、Soul AI 等。
- 当前转向更高级 Agent（ClawBot、OpenClaw 多 Agent 编排）。
- 强调：**快速发货、解决真实问题，而非做 demo**。

### 2.4 数字生命卡兹克

- 日更早报 + 内参；用 AI 打分筛选全网 AI 资讯。
- 方法论：**任何重复 3 遍的事都可 AI 化**。
- 3 月动态：Seedance 2.0、AI 玩梗大会（2000+ 开发者与品牌参与）。

## 四、下一步你应该干什么

### 4.1 若目标是「做 AI 产品 / 一人企业」

1. **先打基础，再上 Agent**
   - 明确：品牌定位、内容支柱、目标客户、落地页、offer。
   - 用 Dan Koe 的 Eden 模板（或类似框架）跑一遍 Brand → Content → Offer。
   - 不要一上来就沉迷 OpenClaw / Claude Code。

2. **选一条变现路径验证**
   - 客户路线：1–2 个 $1000–5000 客户/周，从冷启动开始。
   - 产品路线：需流量，目标 720 人/天访问落地页（2.5% 转化 ≈ 18 单/天 @ $150）。
   - 社交是免费杠杆：目标 10–50K 播放/视频、50 万–100 万曝光/月。

3. **内容一源多用**
   - 1 篇 Newsletter/周 → 1 支 YouTube/周 → 播客 → 帖子 → Reels。
   - AI 做初稿，人做终审和「人类钩子」。

4. **学习 MCP，但控制规模**
   - 在 Cursor 中接入 2–3 个高杠杆工具（如 GitHub、Linear）。
   - 避免一次连太多，导致 token 爆炸和成本失控。

### 4.2 若目标是「紧跟前沿、提升认知」

1. **订阅关键源**
   - Dan Koe Newsletter、AI-Nate 网站、数字生命卡兹克公众号（Wechat2RSS）。
   - 每周花 15–20 分钟扫一遍，标记 1–2 篇精读。

2. **试用 1–2 个新模型/产品**
   - Claude Opus 4.6 / Sonnet 4.6、GPT-5.4、Gemini 3.1 Pro 选一个深度用。
   - 若有开发需求，可关注 Nemotron 3 Super 的开源生态。

3. **识别你的「重复 3 遍」**
   - 列出每天/每周重复做的事（写日报、做竞品分析、写邮件模板等）。
   - 挑 1 件用 AI 或简单自动化替代。

### 4.3 若目标是「技能升级」

| 优先级 | 技能 | 原因 |
|--------|------|------|
| 1 | 内容创作（写+说） | 获客与信任的核心，AI 难完全替代 |
| 2 | Offer 设计与落地页文案 | 转化率决定收入 |
| 3 | Cursor + 基础 MCP | 一人开发效率杠杆 |
| 4 | 工作流自动化（Zapier/Make） | 减少重复操作 |
| 5 | Agent 编排（进阶） | 在 1–4 扎实后再深入 |
