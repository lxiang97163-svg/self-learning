# AI 实战学习手册：软件·场景·落地

> 不是资讯，是可学的、可做的。  
> 目标：用具体软件干具体事，有清单、有流程、能上手。

## 第一部分：软件清单（用什么干吗）

### 1.1 开发与编程

| 软件 | 用途 | 适合谁 | 入门动作 |
|------|------|--------|----------|
| **Cursor** | AI 编程 IDE，写代码、改 Bug、理解项目 | 会写一点代码的人 | 用它开一个现有项目，问「这段代码干什么」再让它帮你重构一小块 |
| **Cursor + MCP** | 让 AI 主动查 GitHub/Linear/数据库，不用你复制粘贴 | 独立开发者、小团队 | 先配 1 个 MCP（见下节），完成 1 次真实任务 |
| **GitHub Copilot** | 行内补全、小段代码生成 | 日常写代码的人 | 在 VS Code 或 Cursor 里装好，边写边用 |
| **Bolt / Windsurf** | 类似 Cursor 的 AI 编程工具，可对比试用 | 想多试几款的人 | 选 1 个和 Cursor 各用 1 周，看哪个顺手 |

### 1.2 内容创作

| 软件 | 用途 | 适合谁 | 入门动作 |
|------|------|--------|----------|
| **ChatGPT / Claude** | 写初稿、改稿、头脑风暴、翻译 | 所有人 | 先写 1 段自己的话，让 AI 扩成 3 段，你再改回自己的语气 |
| **Notion** | 笔记、内容日历、客户管理、知识库 | 一人企业、创作者 | 建 1 个「内容点子」数据库，每有灵感就记一条 |
| **Substack / 竹白 / 即刻** | Newsletter 发布、邮件列表 | 想做长期内容的人 | 先开 1 个号，发 1 篇，不追求完美 |
| **剪映 / CapCut** | 短视频剪辑、字幕、配音 | 想做视频的人 | 用 1 段文字生成 1 个 60 秒口播视频 |
| **Gamma / Beautiful.ai** | PPT、提案、落地页 | 需要对外展示的人 | 输入大纲，让 AI 生成 1 份 10 页 PPT |

### 1.3 自动化与效率

| 软件 | 用途 | 适合谁 | 入门动作 |
|------|------|--------|----------|
| **Zapier** | 跨工具自动化（表单→CRM→邮件） | 有重复流程的人 | 做 1 条：新表单提交 → 发到我邮箱 + 记进 Notion |
| **Make** | 比 Zapier 更复杂的多步骤工作流 | 有进阶需求的人 | 做一个：RSS 更新 → 提取摘要 → 存 Notion |

| **Alfred / Raycast** | 快捷启动、剪贴板、快捷指令 | Mac 用户 | 设置 1 个快捷键打开常用应用 |

### 1.4 一人企业 / 变现

| 软件 | 用途 | 适合谁 | 入门动作 |
|------|------|--------|----------|
| **Gumroad / 爱发电** | 卖数字产品、订阅 | 做课程/模板/会员的人 | 上架 1 个 $5 或 ¥9.9 的小产品 |

## 第二部分：MCP 实操（Cursor 接工具）

### 2.1 什么是 MCP

**MCP = Model Context Protocol**，让 AI 主动去查数据、执行操作，而不是全靠你粘贴。

- 你：**「帮我查这个 repo 最近的 issue」** → Cursor 调 GitHub
- 你：**「把这件事记到 Linear」** → Cursor 调 Linear
- 你：**「查一下数据库里这个用户的记录」** → Cursor 调 Postgres

### 2.2 第一个 MCP：GitHub

**步骤：**

1. 打开 GitHub → Settings → Developer settings → Personal access tokens  
2. 新建 Token，勾选 `repo` 权限  
3. 在 Cursor 中：`Settings` → `Tools & MCP` → `Add new MCP server`  
4. 填写：
   - **Name**: `github`
   - **Type**: `command`
   - **Command**: `npx`
   - **Args**: `["-y", "@modelcontextprotocol/server-github"]`
   - **Env**: `GITHUB_PERSONAL_ACCESS_TOKEN` = 你的 token

5. **完全退出 Cursor 再打开**（必须重启才能生效）

**落地任务：** 在 Cursor 里打开一个 GitHub 项目，对 AI 说：「列出这个 repo 最近 5 个 open 的 issue，并总结每个在讨论什么。」

### 2.3 第二个 MCP：文件系统（本地）

Cursor 自带 `filesystem` MCP，可读写本地文件。

**落地任务：** 对 AI 说：「读取 `outputs` 文件夹下所有 `.md` 文件名，生成一个索引 Markdown 文件。」

### 2.4 建议：先少后多

- 一开始只接 1–2 个 MCP，用熟再用新的  
- 接太多会导致 token 暴涨、变慢、变贵  
- API Key 不要写在 `mcp.json` 里提交到 Git，用环境变量

## 第三部分：可尝试的落地场景

### 场景 1：用 AI 写 1 篇 Newsletter

**流程：**

1. **定主题**：写 1 句话，你要解决读者什么痛点 / 给什么收获  
2. **列大纲**：让 AI 生成 3–5 个小标题，你删改  
3. **写初稿**：把大纲给 AI，说「按这个结构展开，语气像我平时说话」  
4. **人改**：至少改开头、结尾、金句，加 1 个「只有你能写的」例子  
5. **发**：不发完美版，发「够好版」

**Prompt 示例（步骤 3）：**

```
你是我的写作伙伴。我的读者是 [描述]，他们关心 [主题]。

请按以下大纲展开成 800 字左右的初稿，语气要像朋友聊天，避免「首先、其次、总之」这种套路，不要用「在这个充满挑战的时代」这类空话。

大纲：
1. [小标题1]
2. [小标题2]
3. [小标题3]
```

### 场景 2：「重复 3 遍」的 AI 化（数字生命卡兹克思路）

**原则**：同一件事重复做 3 次以上 → 考虑用 AI 或自动化。

**可落地的例子：**

| 你重复做的事 | 可选方案 | 用啥 |
|--------------|----------|------|
| 每天早上查竞品/行业动态 | 定好 RSS 源，Make/Zapier 每日抓取 → 存 Notion，你早上一键看摘要 | Make + Notion |
| 每周写周报 | 固定模板 + 让 AI 根据你的 bullet 扩成段落 | ChatGPT/Claude |
| 写相似的回信/邮件 | 建一个「常见场景 + 模板」文档，让 AI 按场景填充 | Notion + Claude |
| 给每篇文章起标题 | 写好文后，给 AI 5 个候选标题，你选 1 个 | 任意聊天式 AI |
| 把长文缩成推文/短文案 | 「把下面这段话压缩成 3 条推文，每条 280 字以内」 | ChatGPT/Claude |

**本周作业**：列出你「重复 3 遍」的 3 件事，选 1 件用上面的思路试一次。

### 场景 3：4 小时工作制（Dan Koe）

**不是**：每天只干 4 小时  
**而是**：用 4 小时做「必须推进目标」的深度工作，其余时间可做低优先级的。

**实操：**

1. **定 1 个本周唯一重要目标**：比如「完成第一个 Offer 的落地页」  
2. **每天前 4 小时**：只做和这个目标直接相关的事，不开 Slack、不刷手机、不「看看邮件」  
3. **4 小时过后**：可以回复消息、开会、处理杂事  

**工具**：用番茄钟或 Forest 锁手机，或用 Notion 的「今日唯一任务」模板。

### 场景 4：一人企业的「Brand → Content → Offer」跑通

**来源**：Dan Koe Eden 模板思路。

**三步：**

| 步骤 | 做什么 | 产出 |
|------|--------|------|
| **Brand** | 写清：你是谁、帮谁、解决什么问题、和别人有什么不同 | 1–2 句话的定位 + 3 个内容支柱 |
| **Content** | 基于内容支柱，建一个「内容点子库」，每周至少 1 篇长文 + 几条短内容 | 内容日历草案 |
| **Offer** | 定义你的第一个产品/服务：谁买、多少钱、买完能得到什么、为什么非你不可 | 客户画像 + Offer 一句话 + 落地页文案初稿 |

**可配合**：Eden 模板（需注册）或自己用 Notion 建 3 个页面照着填。

**本周作业**：完成 Brand 部分，写出你的 1–2 句定位和 3 个内容支柱。

### 场景 5：用 Cursor 做一个真正的小产品

**思路**：AI-Nate / Pieter 模式——先做一个能用的东西，再谈完美。

**可选方向（挑 1 个）：**

- 一个 **Chrome 插件**：比如「当前页面的内容总结成 3 条 bullet」
- 一个 **简单落地页**：HTML + CSS，介绍你的服务或产品，能收款更好
- 一个 **命令行工具**：比如批量重命名、从某个 API 拉数据到本地
- 一个 **Notion 模板**：比如「周报模板」「内容日历模板」，分享给别人用

**流程**：

1. 用 1 句话描述你要做什么  
2. 在 Cursor 里新建项目，直接对 AI 说这句话  
3. 让 AI 生成初版，你运行、测试、提修改需求  
4. 2 周内能跑起来就发出去，不追求完善

## 第四部分：7 天落地计划（选做）

按你的目标选一条线往下做：

### 路线 A：内容向

| 日 | 动作 |
|----|------|
| 1 | 在 Notion 建「内容点子库」，记 5 个可写的主题 |
| 2 | 用 AI 写 1 篇 500 字短文（按场景 1 流程），发到任意平台 |
| 3 | 选 1 个「重复 3 遍」的事，设计 AI 化方案 |
| 4 | 配好 Cursor 的 GitHub MCP，完成 1 次查 issue 任务 |
| 5 | 写 Brand 定位（1–2 句话 + 3 个内容支柱） |
| 6 | 用 AI 把昨天那篇文变成 3 条推文/短文案 |
| 7 | 复盘：本周哪个动作最有价值，下周继续 |

### 路线 B：产品向

| 日 | 动作 |
|----|------|
| 1 | 决定 1 个小产品方向（落地页 / 插件 / 模板 等） |
| 2 | 在 Cursor 里用 AI 生成初版代码，能跑起来 |
| 3 | 配好 1 个 MCP，用 Cursor 完成 1 次真实开发任务 |
| 4 | 做「重复 3 遍」的 1 件事的 AI 化 |
| 5 | 写完 Offer 初版（谁买、多少钱、能得到什么） |
| 6 | 把产品/落地页发出去，至少 1 个人能访问 |
| 7 | 复盘：有人用吗？反馈是什么？ |

### 路线 C：效率向

| 日 | 动作 |
|----|------|
| 1 | 列出 5 件你「重复 3 遍」的事 |
| 2 | 选 1 件，用 Zapier 或 Make 做 1 条自动化 |
| 3 | 试用 Cursor + 1 个 MCP，完成 1 次任务 |
| 4 | 建一个 4 小时工作制日计划模板，用 1 天 |
| 5 | 用 AI 写 1 篇工作相关的短文（周报/总结/教程） |
| 6 | 整理你的常用 Prompt，存进 Notion 或 Cursor Rules |
| 7 | 复盘：省了多久时间？哪里可以再自动化？ |

## 第五部分：Prompt 与规则库（可直接用）

### 5.1 写作

```
角色：你是我的写作伙伴，语气像朋友聊天，不说空话套话。
任务：根据下面的大纲展开成 [字数] 字初稿。
要求：不用「首先、其次、总之」「在这个充满挑战的时代」；每段有 1 个具体例子或类比。
禁止：AI 腔、堆砌形容词。
```

### 5.2 压缩/改写

```
把下面这段话：
1. 压缩成 3 条推文（每条 280 字以内）
2. 改成更口语化的 1 分钟口播稿
3. 提炼成 3 个 bullet 要点
```

### 5.3 Cursor / 代码

在项目 `.cursor/rules` 或 `.cursorrules` 里加：

```
- 写注释用中文
- 先解释你要做什么再写代码
- 不要一次改太多文件，先改最关键的 1 个
- 有 API Key 时用环境变量，不要写在代码里
```

### 5.4 「重复 3 遍」自查

```
我经常重复做的事：[列出 3 件]
每件事大概花多久：[填]
有没有现成工具能替代：（搜索/问 AI）
我的选择：先做 [选 1 件] 的 AI 化，方案是 [写 1 句]
```

## 第六部分：避坑

| 坑 | 正确做法 |
|----|----------|
| 一上来就玩 Agent / OpenClaw | 先把 Brand、Content、Offer 打牢，再上复杂工具 |
| AI 生成直接发 | 至少改开头、结尾、金句，加你自己才知道的例子 |
| 接一大堆 MCP | 先 1–2 个用熟，再扩展 |
| 追求完美再发布 | 先发「够好版」，根据反馈迭代 |
| 只囤课不练 | 每周至少做 1 件本手册里的「落地动作」 |
