# 校准脚本与竞价量纲（v2 · 完整版）

## 1. `calc_calibration.py` 命令

```bash
python "/home/linuxuser/cc_file/jumpingnow_all/pipeline/calc_calibration.py" --end-date YYYYMMDD --trade-days 30
```

- `YYYYMMDD`：已收盘的最近一个交易日（与复盘表日期一致）。
- **增量缓存**：样本持久化到 `outputs/cache/calibration_cache.json`。首次或换机无缓存会全量回溯（较慢）。
- **每日复盘**：若本次 `--end-date` 正好是「上一缓存窗口末端」的**下一交易日**，只拉取 **1 个新交易日**，滑窗丢掉最旧一日，再汇总中位数。
- 同一 `end-date` 与参数已跑过、窗口一致 → **直接读缓存**，不再逐日请求 API。
- **异常/需重算**：窗口参数变更、`--trade-days` 变更、与 `--include-yizi` 混用导致不一致、隔多日无法滑窗衔接 → 自动**全量回退**并重写缓存。强制重建：`--rebuild-cache` 或 `--no-cache`。

### 脚本统计定义（「30 日实战经验」含义）

- 回溯过去 **30 个交易日**。
- 样本：每日 `limit_list_d` 涨停池（U），按当日 **limit_times（板数）** 分桶。
- **竞价涨幅**：`(stk_auction 价 - 昨收) / 昨收 × 100`。
- **有收益**：当日日线 **收盘 > 开盘**（开盘价买入简化假设）。
- **剔除**竞价涨幅 ≥ 9.5% 的一字样本（可操作口径；需纳入时加 `--include-yizi`）。

### 写入速查的规则

- 按板数区间取**中位数**；写入速查时**仅写百分比**（如 `+3.61%`）；样本数、回溯窗口**只更新**本文件「当前校准值」段，**不写进**速查正文。
- **某桶有效样本为 0**：黄金底线兜底——2～3 板 **+2.5%**、4 板+ **+3.5%**；首板/低吸可 **3%**。

### 与经验库的关系

- **经验库「竞价阈值校准记录」**仍用于验证与失误归档，**不再作为**「当前校准值」取数来源；对照旧算法：`python calc_calibration.py --legacy`。

## 3. 竞价条件量纲规范（执行手册与速查必须遵守）

**核心原则**：执行手册「建议」列和速查的竞价触发条件，统一使用 **`竞价成交额（stk_auction.amount）`** 作为量纲，**不使用封单额**作买入触发器。

### 量纲说明

| 指标 | API 来源 | 用途 | 典型量级 |
|---|---|---|---|
| `竞价成交额` | `stk_auction.amount`（÷1e8 换算亿） | **开盘前可观测，作为买入触发器** | 0.1亿～3亿 |
| `竞价涨幅` | `(stk_auction.price - pre_close) / pre_close × 100` | 辅助判断方向 | -3%～+10% |
| `量比` | `stk_auction.volume_ratio` | 辅助判断成交活跃度 | 1～10 |
| `封单额` | `kpl_list.fd_amount`（÷1e8） | **仅用于复盘表强度历史**，不用于速查触发条件 | 1亿～80亿 |

### 竞价条件写法（相对逻辑，禁止死板绝对值）

| 写法 | 示例 | 是否可用 |
|---|---|---|
| ✅ 两市第一/前排 | `竞价成交额两市第一`、`竞价成交换手两市前排` | 可用 |
| ✅ 竞价回流（相对） | `今日≥昨日×1.0` / `×1.2`、`×0.5～0.8`；**仅**首板/轮动观察可写 `×0.3` | 可用，须与板位匹配 |
| ✅ 竞价涨幅 | `竞价涨幅≥5%` | 可用 |
| ✅ 封板状态 | 风标（封死=情绪温度计） | 见下文「最高板封死 vs 分歧」 |
| ❌ 量比 | `量比≥20` | **不可用**（竞价阶段不显示） |
| ❌ 死板绝对值 | `竞价成交额≥0.5亿` | **禁止** |

### 竞价成交额相对逻辑（强制）

**回流定义**：今日竞价 vs 上次峰值（昨日竞价或近期峰值）。

**速查「买入条件」列必须写清昨日具体亿数**，禁止只写「×0.3」而无亿数、禁止 4 板+ 主门槛只用 ×0.3。

1. **正确示例（4板+）**：`今日竞价≥0.48亿（昨日0.48亿×1.0）；若竞价涨幅≥5%则≥0.58亿（×1.2）`
2. **正确示例（2～3板）**：`今日竞价≥0.42亿（昨日0.42亿×1.0）`
3. **正确示例（首板/轮动试错）**：`今日竞价≥0.05亿（昨日0.15亿×0.3）`——**仅**此类场景可用 0.3

### 分位系数与黄金区间

- **2-3 板**：涨幅门槛经验底线 **+2.5%**；**4 板+**：**+3.5%**（与脚本兜底一致时以脚本为准）。
- **高位连板（≥4板）**：竞价涨幅 ≥ 5% → 系数 **1.2**；涨幅 < 5% → **1.0**。标的高度 ≥ 5 板且竞价涨幅 ≥ 7% 但成交额未达 1.2 倍 → 标注 **「❌盈亏比极差，严禁追高」**。
- **首板/轮动低吸**：可选用 **0.3～0.5** 为最低试错档；前排或强确认仍用 **×1.0**。

### 气质转弱

- 「封单大幅萎缩（昨日 50% 以下）」且「开板次数 >3」→ 气质转弱。
- **主线接力位** → 列入「❌ 不做」。
- **轮动低吸位**且为历史龙一 → 可列「👀 低吸候选」，买入条件须 **1.0 系数（弱转强确认）**，**严禁 0.3**。

### 最高板 / 小盘高位连板（5板+）

| 竞价状态 | 速查写法 |
|---------|-----------|
| **封死**（≈10% 一字） | **做风标，不设买入条件** |
| **有分歧** | 竞价回流（今日≥昨日×**1.0**，小盘可用×0.8）+ 涨幅≥校准值；**勿**默认 ×0.3 |

| 板数 | 错误写法 | 正确写法 |
|:---:|---|---|
| 5板+ 小盘 封死 | 竞价成交额≥2亿、量比≥20 | 风标，不设买入 |
| 5板+ 小盘 分歧 | 成交额≥0.3亿 死板 | 两市前排或今日≥昨日×**1.0～1.2** + 涨幅≥校准值 |
| 4板 正常盘 | 成交额≥0.5亿 死板 | 两市前排或今日≥昨日×**1.0** + 涨幅≥校准值 |

### 速查卡竞价条件参考表

| 股票特征 | 竞价强 | 竞价中 | 竞价弱 |
|---|---|---|---|
| 高位连板（≥4板） | 两市前排或今日≥昨日×**1.2**（涨幅≥5%时）/×**1.0** + 涨幅≥校准值A | 今日≥昨日×**0.5～0.8** | 今日显著低于昨日×**1.0** 或 涨幅<0% |
| 中位板（2~3板） | 两市前排或今日≥昨日×**1.0** + 涨幅≥校准值B | 今日≥昨日×**0.5～0.8** | 同上 |
| 首板/轮动低吸 | 两市前排或今日≥昨日×**1.0** + 涨幅≥校准值C | 今日≥昨日×**0.3～0.5**（试错档） | 今日＜昨日×**0.3** 且无前排 |

### 涨幅校准值说明（进化规则）

- **涨幅校准**：结构性规律，用全量命中样本取中位数（与 `calc_calibration` 一致）；速查只写数字如「涨幅≥+5.5%」。
- **成交额**：已改为相对逻辑（今日≥昨日×系数、两市前排），弱化绝对亿数依赖。
- 经验库「竞价阈值校准记录」中失误类型=「命中」的记录可作补充对照；**主口径以本文件第二节 + 脚本输出为准**。

## 4. 何时更新「竞价阈值参考量级」类默认值

当经验库中同一板数区间的**校准建议 ≥5 条且方向一致**时，可在本包或团队约定的文档中更新对应参考行（与旧版逻辑一致）。
